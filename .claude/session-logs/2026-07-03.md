# 2026-07-03

Continued from a prior session where Phases 1-6 were already built. This session finished Phase 7
(Polish), added a couple of small features, set up local git, and queued up backlog items.

## Bug fixes
- Undo-error message on the kiosk fell back to a generic "Something went wrong" instead of explaining
  that the points had already been redeemed — the check relied on an exact byte-for-byte match against
  the database's raw exception text (including an em dash), which had drifted. Switched to a robust
  substring match and reworded the message to explain why. Applied the same fix to the redemption-modal
  "insufficient points" message for consistency.

## Phase 7: Polish (now complete)
- Audited every route/component for loading/empty/error states — all already present.
- Added `confirm()` guards before archiving a chore or kid (previously a single misclick permanently
  removed one with no confirmation and no restore UI).
- Bumped the kiosk undo button from 24px to 32px (touch target).
- Added a kid-color-tinted background to each kid's kiosk section (~15% opacity of their own avatar
  color), tuned live based on feedback (too subtle -> good).
- Fixed cross-device emoji centering (previously deferred) by switching from native OS emoji fonts to
  bundled Twemoji SVGs:
  - `scripts/copy-emoji-assets.mjs` vendors `@twemoji/svg` into `public/emoji/` via a new `postinstall`
    script (gitignored, regenerated on install).
  - `src/lib/twemoji.ts` resolves any glyph to its bundled SVG via `@twemoji/parser`.
  - `src/components/Emoji.tsx` is the new shared render component.
  - Applied to `KidAvatar`, kiosk chore cards, Chores/Dashboard list prefixes, and the `EmojiPicker`
    itself (custom `getEmojiUrl` pointing at the local bundle instead of its CDN default).
  - Follow-up: the "Assigned to" dropdown in `ChoreForm` still showed a native-font emoji because native
    `<option>` elements can't render `<img>`. Replaced it with a new custom `KidSelect` component that
    renders `KidAvatar` per row, matching everywhere else.

## New features
- Redemption history: `useRedemptionHistory` hook + `RedemptionHistory` modal, reachable from a new
  "History" button on the kiosk header. Reads existing `point_transactions` rows, no schema change
  needed. `useRedeemPoints` now invalidates it on redemption.
- PWA manifest (`public/manifest.webmanifest`, `display: standalone`) so "Add to Home Screen" on the
  Android tablet launches with no browser chrome — the real kiosk fullscreen fix.
- Fullscreen API toggle button (`src/hooks/useFullscreen.ts`) on the kiosk header as a same-tab fallback
  (requires a tap each time; browser security restriction, can't auto-trigger or persist).
